## Фаза 1: Исследование и обзор литературы

### Что изучить

#### 1.1 Базовые архитектуры распознавания речи (Audio-only ASR)

- **CTC** (Connectionist Temporal Classification) - механизм выравнивания последовательностей без явной разметки по времени
- **RNN-Transducer (RNN-T)** - авторегрессионная модель для потокового распознавания
- **LAS** (Listen, Attend and Spell) - архитектура «энкодер-декодер» с вниманием
- **Conformer** - гибрид свёрток и трансформеров, один из лучших энкодеров для речи
- **Whisper** (OpenAI) - большая модель, обученная на 680 тыс. часов размеченных данных
- **Wav2Vec 2.0** и **HuBERT** (Meta) - самообучающиеся модели на неразмеченных аудиоданных

#### 1.2 Визуальное распознавание речи (чтение по губам)

- **LipNet** (2016) - первая end-to-end модель для чтения по губам на уровне предложений
- Визуальные признаки из кропов лица с помощью ResNet-18 и 3D-свёрток
- Понимание того, какую информацию несут движения губ и чем они дополняют аудио

#### 1.3 Аудио-визуальное распознавание речи (AVSR) - ключевая тема

- **AV-HuBERT** (Meta, 2022) - state-of-the-art самообучающаяся модель AVSR
- **Auto-AVSR** (Meta, 2023) - развитие AV-HuBERT с автоматической разметкой
- **RAVEn** - робастная аудио-визуальная модель
- **Стратегии слияния модальностей:** раннее слияние (early fusion), позднее слияние (late fusion), слияние через перекрёстное внимание (cross-attention fusion)

#### 1.4 Потоковое/реальное время ASR

- Чанковое внимание (chunked attention), блочные потоковые трансформеры
- RNN-T как естественно потоковая архитектура
- Адаптации Whisper для стриминга (whisper.cpp, faster-whisper)
- Компромисс между задержкой и точностью

### Ключевые статьи для чтения

2. Ma et al., «Auto-AVSR: Audio-Visual Speech Recognition with Automatic Labels» (2023)

### 2.3 Пайплайн предобработки данных

**Для видео:**
```
Сырое видео → Детекция лица → Вырезание области губ (ROI) → Серое изображение 96×96 пикселей → Последовательность кадров
```

- Для детекции лица и извлечения ключевых точек - использовать **MediaPipe Face Mesh**

- Вырезать область, масштабировать до 96×96, перевести в оттенки серого
- На выходе: тензор размерности (T_видео, 1, 96, 96), где T_видео - количество кадров

**Для аудио:**
```
Сырое аудио → Ресэмплинг до 16 кГц → Нормализация → Лог-мел спектрограмма (80 полос)
```

- Размер окна FFT: 512 сэмплов
- Шаг окна (hop): 160 сэмплов (10 мс)
- Длина окна: 400 сэмплов (25 мс)
- Количество мел-полос: 80
- На выходе: тензор размерности (80, T_аудио)

### 3.2 Вариант A: Построение с нуля (образовательный, рекомендуемый для понимания)

#### Видео-энкодер

1. **Visual part:**
   - 3D-свёртка (Conv3D) для захвата коротких временных паттернов в последовательности кадров губ
   - Вход: (B, T, 1, 96, 96) - батч серых кадров губ
   - После 3D-свёртки: пространственные признаки для каждого кадра

2. **backbone:**
   - Модифицированный ResNet-18, применяемый покадрово для извлечения пространственных признаков
   - На выходе: вектор размерности 512 на каждый кадр

3. **Временной энкодер:**
   - Линейная проекция 512 → d_model (например, 512)
   - Стек из 6 слоёв Transformer Encoder для моделирования временных зависимостей
   - На выходе: (B, T_видео, d_model)

#### Аудио-энкодер

1. **Субдискретизация (Subsampling):**
   - Две 2D-свёрточные слоя с шагом 2 для уменьшения временной размерности в 4 раза
   - Вход: лог-мел спектрограмма (B, 80, T_аудио)

2. **Временной энкодер:**
   - Линейная проекция в d_model
   - Стек из 12 слоёв Transformer Encoder (или Conformer для лучшего качества)
   - На выходе: (B, T_аудио/4, d_model)

#### Модуль слияния (Fusion)

Три варианта (рекомендуется реализовать минимум два для сравнения):

1. **Конкатенация (Late Fusion):**
   - Интерполировать видео-признаки до длины аудио-признаков
   - Конкатенировать по размерности признаков: (B, T, 2·d_model)
   - Линейная проекция обратно в d_model

2. **Перекрёстное внимание (Cross-Attention Fusion) - рекомендуется:**
   - Аудио-признаки «смотрят» на видео через механизм внимания (audio-to-video attention)
   - Видео-признаки «смотрят» на аудио (video-to-audio attention)
   - 4 слоя перекрёстного внимания с FFN и нормализацией
   - Финальная конкатенация и проекция

3. **Раннее слияние (Early Fusion):**
   - Конкатенация сырых признаков на уровне входа в общий энкодер
   - Один общий Transformer обрабатывает обе модальности

#### Декодер / Выходной слой

- **CTC-голова** (самый простой вариант, хорошо подходит для реального времени):
  - Линейный слой d_model → размер_словаря
  - Обучение с CTC-функцией потерь
  - Декодирование жадным алгоритмом или beam search

- **RNN-T** (более подходит для стриминга, но сложнее в реализации)

- **Transformer-декодер** (лучшее качество, но менее подходит для реального времени)

### 3.3 Вариант B: Дообучение предобученных моделей (практичный, быстрые результаты)

**Рекомендуемый подход для курсовой:**

1. **Аудио-ветка:** Взять предобученный Whisper (small или medium) или Wav2Vec 2.0 и заморозить / частично дообучить энкодер
2. **Видео-ветка:** Построить свой видео-энкодер (ResNet-18 + Transformer) или взять визуальный фронтенд из Auto-AVSR
3. **Слияние:** Добавить свой модуль перекрёстного внимания между выходами аудио- и видео-энкодеров
4. **Декодер:** Использовать декодер Whisper или CTC-голову поверх слитых признаков

**Альтернатива:** Склонировать репозиторий **Auto-AVSR** (https://github.com/mpc001/auto_avsr) и дообучить на своих данных.

## Фаза 5: Пайплайн инференса в реальном времени

### 5.1 Архитектура потоковой обработки

```
Веб-камера (25 FPS) ──→ Детекция лица ──→ Кроп губ ──→ Буфер (чанки по 0.5 сек)──┐
                                                                                       ├──→ Слияние → Декодирование → Текст
Микрофон (16 кГц) ──→ Аудио-буфер (чанки по 0.5 сек) ──→ Мел-спектрограмма ────────┘
```

### 5.2 Компоненты реального времени

#### Захват аудио
- Использовать библиотеку **sounddevice** для потокового захвата с микрофона
- Настроить callback-функцию, которая записывает приходящие сэмплы в буфер
- Частота дискретизации: 16 кГц, моно

#### Захват видео
- Использовать **OpenCV** для захвата с веб-камеры
- Запустить в отдельном потоке (threading), чтобы не блокировать основной цикл
- На каждом кадре: детекция лица → извлечение области губ → добавление в буфер
- Целевая частота: 25 FPS

#### Основной цикл инференса
1. Дождаться накопления чанка в обоих буферах (аудио и видео по 0.5 секунды)
2. Предобработать: аудио → мел-спектрограмма, видео → тензор кадров губ
3. Прогнать через модель (без вычисления градиентов)
4. Декодировать выход CTC жадным алгоритмом
5. Вывести текст на экран
6. Сдвинуть буферы с перекрытием (overlap ~100 мс для непрерывности)

#### Обработка граничных случаев
- Что если лицо не обнаружено? → Использовать только аудио-ветку (модель должна это уметь благодаря модальному dropout при обучении)
- Что если аудио тихое / тишина? → Не выдавать транскрипцию, ждать активности (Voice Activity Detection)
- Что если задержка слишком велика? → Уменьшить размер чанка или модели

### 5.3 Оптимизация модели для реального времени

| Метод | Описание | Ожидаемое ускорение |
|-------|----------|---------------------|
| **Смешанная точность (FP16)** | Инференс в половинной точности | ×1.5–2 |
| **torch.compile** | Компиляция графа PyTorch 2.x | ×1.3–2 |
| **TorchScript** | Трассировка/скриптирование модели | ×1.2–1.5 |
| **ONNX Runtime** | Экспорт в ONNX, инференс через ONNX Runtime с CUDA | ×1.5–3 |
| **TensorRT** | Оптимизация NVIDIA для максимальной скорости | ×2–5 |
| **Квантизация (INT8)** | Уменьшение точности весов до 8 бит | ×1.5–2 |
| **Уменьшение модели** | Pruning, knowledge distillation | Зависит от степени |

**Рекомендуемый порядок:** сначала FP16 + torch.compile, затем ONNX Runtime. TensorRT - если нужна максимальная скорость.

### 5.4 Критерий «реального времени»

- **RTF (Real-Time Factor)** = время обработки / длительность аудио
- RTF < 1.0 означает, что система работает быстрее реального времени
- **Целевая задержка:** < 500 мс от момента произнесения до появления текста
- Измерять на целевом оборудовании (GPU, на котором будет запускаться демо)

## Фаза 6: Оценка и эксперименты

### 6.1 Основные метрики

| Метрика | Описание | Инструмент |
|---------|----------|------------|
| **WER** (Word Error Rate) | Доля ошибок на уровне слов | библиотека jiwer |
| **CER** (Character Error Rate) | Доля ошибок на уровне символов | библиотека jiwer |
| **Задержка** (Latency) | Время от начала обработки чанка до получения текста (мс) | замер времени |
| **RTF** (Real-Time Factor) | Отношение времени обработки к длительности аудио | вычисление |
| **Пропускная способность** | Число обработанных секунд аудио в секунду | замер |

### 6.2 Необходимые эксперименты и аблации

| № | Эксперимент | Что измеряем |
|---|-------------|-------------|
| 1 | **Аудио-only бейзлайн** | WER на чистых и зашумлённых данных (только аудио-ветка) |
| 2 | **Видео-only бейзлайн** (чтение по губам) | WER только по видео (без аудио) |
| 3 | **Аудио + Видео (ваша модель)** | WER на чистых и зашумлённых данных |
| 4 | **Робастность к шуму** | WER при SNR = 0, 5, 10, 15, 20 дБ |
| 5 | **Аблация по методу слияния** | Сравнение: конкатенация vs. cross-attention vs. early fusion |
| 6 | **Бенчмарк задержки** | RTF, мс на чанк, на разном оборудовании |
| 7 | **Деградация при отсутствии модальности** | Что будет если убрать видео? Если убрать аудио? |
| 8 | **Влияние размера модели** | WER и RTF для маленькой / средней / большой модели |
